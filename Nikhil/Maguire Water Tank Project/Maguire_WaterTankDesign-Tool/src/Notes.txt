Maguire Water Tank Design Tool - Codebase Analysis
======================================================

Overview:
---------
This is a Windows Forms (WinForms) application built with .NET 8.0, designed for the structural engineering of water tanks. It follows a modular, segment-based design approach where users build a tank by stacking different sections (base, cylinder, riser, head) and applying various loads for analysis.

1. Architecture & Core Components:
----------------------------------
* Program.cs: The entry point. It sets up DPI awareness, initializes the DIContainer, and launches the application.
* DIContainer.cs: A simple Dependency Injection container that manages service instances like AppState and the Database Context.
* AppState.cs: Maintains the global state of the application (e.g., current tank type, selected project) and persists it to 'appstate.json'.
* WaterTankDbContext.cs & Migrations: Uses Entity Framework Core with SQLite. Each project has its own SQLite database file ('project_data.db') to store segment properties, material data, and tank configurations.

2. UI & Workflow:
-----------------
* TankTypeSelectionForm.cs: The first screen where users choose between 'Single Pedestal' and 'Multi-Column' tanks.
* StartupForm.cs: Handles project management—creating new projects, opening existing ones, and setting project-level metadata.
* WaterTank.cs: The main workspace. It coordinates the design process, allowing users to add segments, materials, and loads. It provides a visual representation of the tank.
* TankTypeCard.cs: A custom UI component used in the selection form to represent different tank types.

3. Core Logic Folders:
----------------------
* Segments/: This is the heart of the design. A tank is composed of 'Segments'.
    - SegmentDialogBox.cs: Logic for adding/editing segments, ensuring vertical continuity (matching heights between segments).
* MultiColumn/: Dedicated module for multi-leg tank designs.
    - Contains specialized segment logic specifically for tanks with multiple support columns.
* Solver/ & Solver Equation/: The calculation engine.
    - Equations.cs: Contains mathematical formulas for cylinders, cones, and tank heads.
    - Solver Output.cs: The primary processor that calculates projected areas, centroids, weights, and wind/seismic forces. It validates results against allowable stresses.
* Load/: Handles environmental and structural loads.
    - Covers Wind, Seismic, Snow, Dead, Live, and Water loads.
    - Constants are often sourced from building codes based on risk categories.
* Constants/:
    - ConstantsClass.cs: A central repository for engineering constants, material properties, and regulatory coefficients used across the solver.
    - AppEnums.cs: Defines standardized types for tank components and load categories.
* Designer Notes/:
    - DesignerNotes.cs & NotesManager.cs: A system for engineers to attach technical annotations or design justifications to specific parts of the tank project.
* Entity/: Contains the data models (POCO classes) that represent the tank components in the database.
* Tanks/: Manages tank templates and predefined configurations.
    - tanks.json & MultiLeg-Tanks.json: JSON files acting as a database for standard tank dimensions.
* Foundation Design/ & Foundation Properties/:
    - Implements engineering logic for anchor bolts, base plates, and foundation stability.
* Custom Design Control/:
    - TankDesign.cs: Intended for dynamic GDI+ drawing of the tank structure (though some parts currently use static images).

4. Output & Export:
-------------------
* Output/: Responsible for generating reports.
    - ExportFabricationOutput.cs: Uses the ClosedXML library to export detailed fabrication parameters to Excel. It maps internal C# entities to over 100 specific fabrication fields.

5. Key Connections & Interactions:
----------------------------------
1. Selection: User selects a tank type -> AppState stores the choice.
2. Project: StartupForm creates a project folder and initializes the SQLite DB via DbContextProvider.
3. Design: In WaterTank.cs, the user adds Segments (stored in DB). Each segment addition triggers a UI refresh and updates the total height.
4. Loading: User inputs load parameters (Wind speed, Seismic zone) -> Stored in AppState/DB.
5. Solving: The Solver reads all segments and loads from the DB, runs the Equations, and displays results in 'Solver Output'.
6. Export: Once the design is validated, the Output module aggregates data for fabrication.

PROJECT DOMAIN:
Structural engineering CAD tool for designing elevated water tanks (Single Pedestal and Multi-Leg). It calculates structural integrity based on environmental loads.

TECH STACK:
- UI: Windows Forms (WinForms) on .NET 8.0 (C#)
- Database: SQLite per-project file (`project_data.db`) managed via Entity Framework Core (`WaterTankDbContext.cs`).
- Export: ClosedXML for generating Excel fabrication output.

CORE ARCHITECTURE & STATE:
- Entry/State: `Program.cs` bootstraps the app. `DIContainer.cs` handles simple dependency injection. `AppState.cs` holds global state (current tank type, active project path) and persists it locally.
- Project Structure: User creates a project (via `StartupForm.cs`). This creates a local folder with its own SQLite database.
- Main Workspace: `WaterTank.cs` is the central form orchestrating the visual layout and user interactions.

ENGINEERING DOMAIN LOGIC:
1. Segments (`Segments/`): The tank is built bottom-up via distinct segments. `SegmentDialogBox.cs` enforces vertical continuity (HeightFinal of one segment must match HeightInitial of the next).
2. Load Calculations (`Load/`): Environmental factors (Wind, Seismic, Snow, Dead/Live/Water load). Constants are pulled from `Constants/ConstantsClass.cs` (based on building codes/risk categories).
3. The Solver (`Solver/` & `Solver Equation/`): The brain of the app. `Equations.cs` contains raw geometry and physics math. `Solver Output.cs` consumes segments and loads, calculates projected areas/centroids/weights, and validates against allowable stresses.
4. Foundation (`Foundation Design/`): Handles base plate and anchor bolt engineering.

UI PATTERNS:
- Relies heavily on static images (e.g., `Resources/`) combined with overlaid labels for visual feedback, though `Custom Design Control/TankDesign.cs` indicates intentions for dynamic GDI+ drawing.
- MultiColumn support (`MultiColumn/`) is separate and seemingly less mature than the Single Pedestal workflow.

DATA FLOW:
User inputs (Dimensions, Loads) -> Saved to local SQLite -> Solver reads DB & Calculates Stresses -> Results displayed in `Solver Output` -> `Output/ExportFabricationOutput.cs` maps entities to Excel for manufacturing.
